# RFC (Request For Comments)

## ¿Qué es?

El RFC documenta una propuesta técnica para resolver un problema definido previamente en un PRD.

Describe la solución propuesta, alternativas consideradas, riesgos y estrategia de implementación.

---

## ¿Cuándo usarlo?

Utilizar cuando:

* La solución requiere diseño técnico.
* Existen múltiples alternativas posibles.
* Se espera impacto en varios sistemas o equipos.
* Se necesita consenso técnico antes de desarrollar.

No suele ser necesario para cambios simples de bajo impacto.

---

## ¿Quién participa?

### Responsables

* Tech Lead
* Arquitecto de Software

### Participantes

* Backend Developers
* Frontend Developers
* QA
* Engineering Manager

---

## ¿Qué entregable produce?

* Solución propuesta.
* Alternativas evaluadas.
* Riesgos identificados.
* Impactos técnicos.
* Estrategia de implementación.
* Épicas técnicas.

---

## ¿Cómo ayuda a la IA?

Permite que una IA:

* Comprenda la arquitectura propuesta.
* Evalúe riesgos.
* Sugiera mejoras técnicas.
* Genere ADRs.
* Genere Technical Briefs.
* Ayude a elaborar historias de usuario técnicas.

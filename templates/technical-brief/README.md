# Technical Brief

## ¿Qué es?

El Technical Brief es un documento de preparación para la implementación.

Su objetivo es reducir incertidumbre antes de estimar y desarrollar una funcionalidad.

Describe los componentes afectados, dependencias, validaciones, pruebas e impactos técnicos.

---

## ¿Cuándo usarlo?

Utilizar cuando:

* Se inicia el refinamiento técnico de una historia.
* Es necesario estimar esfuerzo.
* Existen múltiples sistemas involucrados.
* Se requiere identificar riesgos antes del desarrollo.

---

## ¿Quién participa?

### Responsables

* Backend Developers
* Frontend Developers

### Participantes

* QA
* Tech Lead

---

## ¿Qué entregable produce?

* Dependencias identificadas.
* APIs afectadas.
* Eventos involucrados.
* Impactos en base de datos.
* Validaciones.
* Riesgos técnicos.
* Estrategia de pruebas.
* Estimación preliminar.

---

## ¿Cómo ayuda a la IA?

Permite que una IA:

* Comprenda el alcance técnico real.
* Detecte dependencias ocultas.
* Sugiera casos de prueba.
* Identifique riesgos.
* Ayude a estimar esfuerzo.
* Genere tareas técnicas de implementación.

Es una de las mejores fuentes de contexto para asistentes de IA durante el desarrollo diario.

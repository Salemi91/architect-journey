# PRD (Product Requirements Document)

## ¿Qué es?

El Product Requirements Document (PRD) describe el problema de negocio que se desea resolver, el valor esperado para los usuarios y los requisitos funcionales y no funcionales de una iniciativa.

Su objetivo es alinear a producto, negocio y tecnología antes de iniciar el diseño técnico.

---

## ¿Cuándo usarlo?

Utilizar cuando:

* Se propone una nueva funcionalidad.
* Se lanza un nuevo producto.
* Se realiza una mejora significativa sobre un producto existente.
* Es necesario alinear a múltiples áreas antes de iniciar el desarrollo.

No suele ser necesario para cambios menores o correcciones simples.

---

## ¿Quién participa?

### Responsables

* Product Manager
* Product Analyst

### Participantes

* Engineering Manager
* Stakeholders
* Tech Lead
* Usuarios de negocio

---

## ¿Qué entregable produce?

* Problema a resolver.
* Objetivos de negocio.
* Requisitos funcionales.
* Requisitos no funcionales.
* Métricas de éxito.
* Alcance y restricciones.

---

## ¿Cómo ayuda a la IA?

Permite que herramientas de IA comprendan:

* El contexto de negocio.
* Los objetivos del producto.
* Los actores involucrados.
* Las funcionalidades esperadas.

A partir de un PRD una IA puede ayudar a generar RFCs, historias de usuario, casos de prueba y documentación técnica inicial.
